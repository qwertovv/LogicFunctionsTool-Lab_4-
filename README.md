# Logic Functions Tool

## Описание проекта

Logic Functions Tool - это приложение на C# WPF для работы с булевыми функциями. Программа предоставляет удобный графический интерфейс для анализа логических функций, построения таблиц истинности, генерации нормальных форм и проверки эквивалентности булевых выражений.

**Основной функционал:**
- Задание булевых функций по номеру или по формуле
- Построение полных таблиц истинности
- Генерация ДНФ (дизъюнктивной нормальной формы) и КНФ (конъюнктивной нормальной формы)
- Проверка эквивалентности двух логических функций
- Подсчет "стоимости" формул (литералы, конъюнкты, дизъюнкты)
- Поддержка основных логических операторов: ¬, ∧, ∨, ⊕, →, ≡

## Архитектура проекта

Проект реализован с использованием паттерна MVVM (Model-View-ViewModel) и состоит из следующих компонентов:

### Модели (Models)
- `LogicalFunction` - представляет булеву функцию, содержит методы генерации таблиц истинности
- `TruthTableRow` - хранит одну строку таблицы истинности
- `NormalForm` - содержит информацию о нормальной форме и ее характеристиках

### ViewModel
- `MainViewModel` - основная ViewModel, связывающая представление с бизнес-логикой
- Реализует команды для генерации таблиц, ДНФ/КНФ и проверки эквивалентности
- Содержит свойства для привязки данных в UI

### Утилиты (Utilities)
- `FormulaParser` - лексер и парсер логических формул
- Преобразование инфиксной записи в обратную польскую запись (ОПЗ)
- Конвертация расширенных операторов в базис {¬, ∧, ∨}

### Представление (View)
- `MainWindow.xaml` - главное окно с тремя вкладками
- `HelpWindow.xaml` - окно справки с инструкциями
- Реализованы вкладки: "По номеру", "По формуле", "Сравнение"

### Тесты
- Полное покрытие xUnit тестами всех основных компонентов
- Тесты генерации таблиц истинности, парсинга формул, нормальных форм
- Интеграционные тесты полного workflow

## Системные требования

### Минимальные требования:
- **ОС:** Windows 10 версии 1809 или новее
- **.NET:** .NET 8.0 Runtime
- **Память:** 512 MB RAM
- **Процессор:** 1 GHz или быстрее
- **Место на диске:** 50 MB

### Рекомендуемые требования:
- **ОС:** Windows 10/11 последней версии
- **.NET:** .NET 8.0 SDK (для сборки из исходников)
- **Память:** 1 GB RAM
- **Процессор:** 2 GHz, 2 ядра
- **Разрешение экрана:** 1280×720 или выше

### Зависимости:
- .NET 8.0 Desktop Runtime (для запуска)
- Windows Presentation Foundation (WPF)
- xUnit (для тестов, только при разработке)

## Установка и запуск

### Сборка из исходного кода:
1. Убедитесь, что установлен .NET 8.0 SDK
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/LogicFunctionsTool.git
   ```
3. Перейдите в папку проекта:
   ```bash
   cd LogicFunctionsTool
   ```
4. Соберите проект:
   ```bash
   dotnet build --configuration Release
   ```
5. Запустите:
   ```bash
   dotnet run --project LogicFunctionsTool
   ```

### Готовая сборка:
Скачайте последний релиз, распакуйте архив и запустите `LogicFunctionsTool.exe`

## Использование

1. **Вкладка "По номеру":** Задайте количество переменных и номер функции для генерации таблицы истинности и нормальных форм
2. **Вкладка "По формуле":** Введите логическое выражение используя поддерживаемые операторы
3. **Вкладка "Сравнение":** Проверьте эквивалентность двух функций и найдите контрпример при различии

Поддерживаемые операторы: `!` (not), `&` (and), `|` (or), `^` (xor), `->` (импликация), `=` (эквивалентность)

## Разработка

Проект использует современные практики C# разработки:
- Асинхронное программирование
- Привязка данных через INotifyPropertyChanged
- Командный паттерн для обработки UI событий
- Модульное тестирование с xUnit
- Четкое разделение ответственности по слоям
